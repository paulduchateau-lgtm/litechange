**Step 2: Create DTCG JSON source files in `design-system/tokens/src/`.**

All JSON files MUST use DTCG 2025.10 format with `$type` and `$value` properties. The naming convention drops the `color` category segment per user decision: `--liteops-olive-900` NOT `--liteops-color-olive-900`.

**Step 3: Create Style Dictionary build script `design-system/tokens/build.mjs`.**

The build script must:

1. Import StyleDictionary from 'style-dictionary'
2. Register a custom format `css/themed` that produces TWO selector blocks:
   - `:root, [data-theme="light"] { --liteops-*: light values }`
   - `[data-theme="dark"] { --liteops-*: dark values }`
3. Register a custom format `tailwind/theme` that wraps CSS variables in `@theme { }` block. Only include raw palette + typography + spacing + radius tokens (semantic tokens are consumed via CSS custom properties, not Tailwind utilities)
4. Register a custom format `css/compat` that generates the backward compatibility shim mapping every `--lc-*` variable to its `--liteops-*` equivalent using `var()` references
5. Configure platforms:
   - `css`: outputs `liteops-tokens.css` with themed format, prefix `liteops`
   - `tailwind`: outputs `liteops-tailwind.css` with tailwind/theme format
   - `json`: outputs `liteops-tokens.json` with `json/flat` format
   - `compat`: outputs `lc-compat.css` with css/compat format
6. Call `await sd.buildAllPlatforms()`

The prefix is `liteops` (Style Dictionary adds the `--` and `-` separator automatically).

IMPORTANT: Style Dictionary v5 uses DTCG `$value` and `$type` natively. Use `outputReferences: true` so semantic tokens reference raw tokens via `var()` in the CSS output.

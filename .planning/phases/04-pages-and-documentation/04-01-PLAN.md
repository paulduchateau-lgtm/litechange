**Naming conventions page (naming-conventions.html):**

Create a standalone HTML page documenting the CSS naming system.

Section 1 -- BEM-like classes:
- Pattern: `.liteops-Component--variant` for modifiers
- Pattern: `.liteops-Component__element` for children
- State: `.is-error`, `.is-disabled` on wrapper element
- Show real examples from the design system: `.liteops-Button--primary`, `.liteops-Button--lg`, `.liteops-Hero__heading`

Section 2 -- Token naming:
- Tier 1 (Raw): `--liteops-{scale}-{step}` (e.g., --liteops-olive-900)
- Tier 2 (Semantic): `--liteops-{role}-{variant}` (e.g., --liteops-text-primary)
- Tier 3 (Component): `--liteops-{component}-{property}` (e.g., --liteops-interactive-primary)

Section 3 -- Component structure:
- Atoms: button, badge, input, select, textarea, switch, checkbox, radio, slider, glass-panel, skeleton
- Molecules: kpi-card, alert, form-group, nav-item, breadcrumb, tab-pill, progress-bar, toast
- Organisms: sidebar, topbar, data-table, modal, hero-section, content-section, card-grid, agent-card, pipeline-viz, multi-select

Section 4 -- File naming:
- Components: `design-system/components/{tier}/{name}.html`
- Tokens: `design-system/tokens/liteops-tokens.css`
- Pages: `design-system/pages/{name}.html`

Use code blocks (dark background olive-900, green keywords) for examples per CLAUDE.md architecture doc patterns.
